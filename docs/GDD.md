# 게임 기획서 — PixelSlime

---

## 1. 게임 개요

### 게임 타이틀
**PixelSlime** *(가제)*

### 게임 장르
소셜 성장형 RPG / 캐주얼 라이프 시뮬레이션

### 게임 플랫폼
- 1차 출시: PC (Windows)
- 2차 목표: Mobile (iOS / Android)

### 게임 설명
아무것도 없는 원형 슬라임으로 태어나 세상을 탐험하며 성장해 나가는 픽셀 아트 소셜 RPG.
플레이어는 쿼터뷰(아이소메트릭) 세계를 자유롭게 이동하며, 퀘스트를 수행하고 다른 플레이어와 교류하면서 슬라임을 진화시켜 원하는 종족으로 탄생시킨다.
내 집을 꾸미고, 펫을 키우고, 미니게임에서 경쟁하는 라이프 시뮬레이션 요소도 함께 즐길 수 있다.

### 게임의 목표
- **단기 목표**: 슬라임을 성장시켜 원하는 종족(인간/오크/엘프/수인)으로 진화
- **중기 목표**: 나만의 집을 꾸미고, 펫을 육성하며 세계 내 생활 기반 구축
- **장기 목표**: 다른 플레이어와 교류·경쟁하며 인기 투표 1위 달성

## 2. 게임 시스템 디자인

### 게임 진행 방식

```
[시작] 원형 슬라임 생성
   ↓
[성장] 퀘스트·탐험으로 EXP 획득 → growthStage 상승 (0→5)
   ↓
[진화] growthStage 5 달성 시 종족 선택
   ↓
[라이프] 집 꾸미기 / 펫 육성 / 미니게임 / 인기투표 참여
```

**성장 단계 (growthStage)**

| 단계 | 외형 변화 | 해금 콘텐츠 |
|------|-----------|-------------|
| 0 | 원형 슬라임 | 기본 이동, 맵 탐험 |
| 1 | 눈 생성 | NPC 대화, 퀘스트 수락 |
| 2 | 입 생성 | 펫 교류, 대화 기능 |
| 3 | 팔 생성 | 아이템 획득, 집 꾸미기 |
| 4 | 다리 생성 | 미니게임 참여, 빠른 이동 |
| 5 | 종족 선택 | 종족 전용 스킬·외형 |

### 게임 룰 (게임 규칙)

- **EXP 획득**: 퀘스트 완료, 탐험, 미니게임 참여, 펫 돌보기
- **치장 아이템**: 슬라임 단계별 착용 가능 아이템 존재. 종족 선택 후 추가 슬롯 해금
- **집**: 기본 집 → 업그레이드로 공간 확장. 가구 배치는 자유
- **펫**: 먹이·놀아주기로 친밀도 상승 → 친밀도에 따라 버프 제공
- **인기투표**: 매주 초기화. 다른 플레이어에게 투표받아 랭킹 경쟁
- **미니게임**: 실시간 WebSocket 기반 멀티 미니게임 (상세 기획 별도)

### 게임 인터페이스 (UI/UX)

**HUD (게임 플레이 중)**
```
┌─────────────────────────────────┐
│ [캐릭터 아이콘] LV.3  EXP ████░  │  ← 좌상단
│                                 │
│        [ 쿼터뷰 맵 ]             │
│                                 │
│  [퀘스트]  [집]  [펫]  [인벤]    │  ← 하단 메뉴바
└─────────────────────────────────┘
```

**주요 화면**
- 메인 맵 화면 (쿼터뷰 이동)
- 캐릭터 정보 / 치장 화면
- 집 인테리어 화면
- 펫 관리 화면
- 미니게임 대기/플레이 화면
- 인기투표 랭킹 화면

## 3. 캐릭터 및 시나리오

### 주요 캐릭터 소개

**플레이어 캐릭터 — 슬라임**
- 아무 기억도 없이 이 세계에 떨어진 원형 슬라임
- 탐험과 경험을 통해 자신의 모습을 완성해 나감
- 최종적으로 4개 종족 중 하나로 진화

**종족별 콘셉트**

| 종족 | 콘셉트 | 특징 |
|------|--------|------|
| 인간 (Human) | 균형잡힌 모험가 | 범용 스킬, 높은 적응력 |
| 오크 (Orc) | 강인한 전사 | 물리 공격·방어 특화 |
| 엘프 (Elf) | 자연의 수호자 | 원거리·마법, 펫 친밀도 보너스 |
| 수인 (Beastman) | 야생의 질주자 | 이동속도·탐험 범위 특화 |

**NPC 캐릭터 (예정)**
- 마을 상인: 치장 아이템 판매
- 퀘스트 의뢰인: 각 구역별 퀘스트 제공
- 가이드 슬라임: 튜토리얼 안내

### 스토리 개요 (시놉시스)
> *"어느 날 알 수 없는 빛과 함께 이 세계에 떨어진 슬라임. 아무것도 기억나지 않지만 이 세계의 사람들과 만나고, 모험을 거듭하며 조금씩 자신의 모습을 되찾아 간다. 그리고 마침내 — 나는 누가 될 것인가?"*

### 배경 설정
- 세계관: 픽셀 판타지 세계 **"크로마 월드(Chroma World)"**
- 마을, 숲, 던전, 바다 등 다양한 구역으로 구성
- 각 구역마다 종족 특성이 반영된 환경과 퀘스트 존재

## 4. 레벨 디자인 설계

### 스테이지 레벨 목록

| 구역 | 이름 | 대상 growthStage |
|------|------|-----------------|
| 1 | 시작 마을 (Start Village) | 0~1 |
| 2 | 초록 평야 (Green Plains) | 1~2 |
| 3 | 안개 숲 (Mist Forest) | 2~3 |
| 4 | 황폐한 유적 (Ruins) | 3~4 |
| 5 | 종족 성역 (Tribe Sanctuary) | 4~5 (종족 선택 장소) |

### 각 레벨의 목적과 특징

**1. 시작 마을**
- 목적: 튜토리얼, 기본 조작 습득
- 특징: NPC 가이드, 간단한 퀘스트, 상점 이용법 안내

**2. 초록 평야**
- 목적: 첫 번째 탐험, 펫 만나기
- 특징: 넓은 이동 공간, 야생 펫 출현

**3. 안개 숲**
- 목적: 치장 아이템 수집, 미니게임 해금
- 특징: 시야 제한 연출, 숨겨진 오브젝트

**4. 황폐한 유적**
- 목적: 고난이도 퀘스트, 희귀 아이템 획득
- 특징: 고레벨 NPC, 멀티 미니게임 참여 필요

**5. 종족 성역**
- 목적: growthStage 5 달성 후 종족 선택
- 특징: 4개 종족 지역으로 나뉜 성역, 선택 후 외형 변경

## 5. 게임 그래픽 아트 디자인

### 캐릭터 디자인 (PC/NPC)
- **스타일**: 16×16 또는 32×32 픽셀 아트
- **애니메이션**: 대기/이동/감정표현 (최소 3종)
- **슬라임 단계별**: growthStage 0~5 각각 별도 스프라이트 시트
- **종족 외형**: 종족 선택 후 베이스 스프라이트 교체 + 치장 레이어 오버레이

### 배경 디자인 (BGA)
- **타일 크기**: 64×32px (아이소메트릭 다이아몬드)
- **타일 종류**: 바닥(잔디/돌/모래), 오브젝트(나무/건물/바위), 장식물
- **레이어 구성**:
  ```
  Ground    ← 바닥 타일
  Object    ← 나무, 건물 등 (Y축 정렬)
  Character ← 플레이어, NPC
  UI        ← 머리 위 이름표, 이모티콘
  ```

### 효과 디자인 (Effect)
- 레벨업 파티클 (슬라임 성장 시)
- 종족 선택 연출 (빛 이펙트)
- 감정 이모티콘 (말풍선 형태)
- 아이템 획득 팝업

## 7. 게임 플레이 테스트 계획

### 플레이 테스트 계획

| 단계 | 내용 | 시기 |
|------|------|------|
| Alpha | 핵심 이동·성장 시스템 동작 확인 | Unity 기본 구현 완료 후 |
| Beta (내부) | 서버 연동, API 통신, 데이터 저장 검증 | 서버-클라이언트 연동 완료 후 |
| Beta (외부) | 소수 유저 플레이 테스트, 밸런스 피드백 | 전체 기능 완성 후 |
| RC | 최종 버그 수정, 출시 준비 | 출시 1개월 전 |

**테스트 중점 항목**
- 성장 단계별 퀘스트 EXP 밸런스
- 서버 동시접속 부하 테스트 (Drogon WebSocket)
- 쿼터뷰 이동·충돌 자연스러움
- 집/펫/인벤토리 데이터 동기화 정확성

### 게임 디버깅 계획
- Unity Console 로그 레벨 구분 (Info / Warning / Error)
- 서버 사이드: Drogon 요청/응답 로깅
- MongoDB 데이터 정합성 체크 스크립트
- 클라이언트 에러 → 서버 에러 리포팅 연동 (추후)

## 8. 게임 출시 및 마케팅 계획

### 출시 일정 (예상)

| 마일스톤 | 내용 | 예상 시기 |
|----------|------|----------|
| M1 | Unity 기본 구현 (이동·타일맵·캐릭터) | 개발 시작 후 1개월 |
| M2 | 서버-클라이언트 연동 완료 | M1 + 1개월 |
| M3 | 전체 콘텐츠 구현 (집·펫·미니게임) | M2 + 2개월 |
| M4 | 알파 테스트 | M3 + 2주 |
| M5 | 베타 테스트 (외부) | M4 + 1개월 |
| **출시** | PC (Steam 또는 자체 배포) | M5 + 1개월 |

### 게임 가격 (예상 발매가)
- **기본 플레이**: 무료 (Free to Play)
- **수익 모델**:
  - 치장 아이템 패키지 (코스튬, 집 가구 세트)
  - 펫 희귀 스킨
  - 프리미엄 배틀패스 (시즌제)
- **정책**: Pay-to-Win 요소 없음. 모든 게임플레이 콘텐츠는 무료

### 게임 마케팅 전략

**1단계 — 사전 공개**
- 개발 과정 영상/스크린샷 SNS 공유 (TikTok, Instagram, X)
- 픽셀 아트 캐릭터 디자인 공개로 관심 유도

**2단계 — 베타 모집**
- 디스코드 커뮤니티 개설
- 얼리 액세스 베타 테스터 모집 (한정 인원)
- 베타 참여자 전용 치장 아이템 지급으로 입소문 유도

**3단계 — 출시**
- Steam 페이지 등록 (위시리스트 캠페인)
- 게임 유튜버·스트리머 협업
- 인기투표 시스템을 활용한 커뮤니티 자생적 홍보 유도
